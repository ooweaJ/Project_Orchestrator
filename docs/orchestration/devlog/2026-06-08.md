# 2026-06-08 Devlog

## Expanded Orchestration Interface

- Goal: Update the orchestration standard from the original six-role set to the expanded required core and recommended extensions.
- Changes: Added `STATUS.md`, `RUNBOOK.md`, `SCOPE_GUARD.md`, `README.md`, `devlog/`, and evidence-oriented extension folders to the standard.
- Verification: Markdown files reviewed after editing.
- Problem: The original interface did not clearly separate current state, active task, commands, scope limits, internal logs, and external reports.
- Decision: Treat `reports/` as external-facing output and `devlog/` as the internal black box. Treat AGENTS templates as repository templates first, then promote to a Codex skill after repeated use.
- Next: Implement scanner and dashboard support for detecting this expanded interface.

## Install Command

- Goal: Make the orchestration interface easy to scaffold into another local project.
- Changes: Added `scripts/install-orchestration.mjs`, `npm run orchestration:install`, and default Markdown templates under `docs/orchestration/templates/`.
- Verification: `npm run orchestration:install -- --target . --dry-run` passed.
- Problem: The first dry-run output listed existing directories as created.
- Decision: The installer should skip existing files and folders, and only create missing entries.
- Next: Use this command on real LETHE and SoulLike paths after confirming the target projects.

## Existing Project Migration Prompt

- Goal: Support active projects that already have docs and work history.
- Changes: Added `EXISTING_PROJECT_MIGRATION_PROMPT.md` as an AI handoff prompt for mapping legacy docs into the orchestration interface.
- Verification: Template content reviewed.
- Problem: Empty template installation is not enough for ongoing projects because existing state, decisions, evidence, and scope rules need to be interpreted.
- Decision: Use install command for scaffolding, and use the migration prompt for meaning-preserving conversion.
- Next: Try the prompt on one real project before turning the process into a dashboard action.
