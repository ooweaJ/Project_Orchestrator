어떤 작업

  • 위험도를 실제 행동 단위로 나누는 액션 카테고리를 추가했습니다.
  • 계속 구현, 검증, 정리, push 준비용 Codex 프롬프트를 추가했습니다.
  • 로컬 경로와 파일명을 숨기는 Portfolio Mode를 대시보드에 추가했습니다.
  • 스캔 결과 기반 Discord 보고서 API와 전송 스크립트를 추가했습니다.

진행 내용

  • 예시 경로처럼 존재하지 않는 프로젝트는 blocked로만 분류하고, 문서 누락으로 잘못 세지 않게 조정했습니다.
  • 스냅샷 보고서가 HTML 보고서 출처와 헷갈리지 않도록 Discord footer 문구를 분리했습니다.
  • 브라우저 시각 검증은 런타임 제약이 있어 API와 빌드 검증으로 대체했습니다.

결과

  • npm run build가 통과했습니다.
  • GET /api/report가 한글 요약 보고서 JSON을 반환했습니다.
  • npm run report:discord:snapshot:dry가 스캔 기반 Discord payload를 정상 생성했습니다.
  • 임시 프로젝트로 push/verification 프롬프트 생성까지 확인했습니다.
  • 다음 작업은 실제 LETHE/SoulLike 경로를 연결해서 진짜 프로젝트 상태를 검증하는 것입니다.
