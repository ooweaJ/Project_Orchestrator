개발 문서 플러그인 설치 흐름

2026-06-08 작업 완료

어떤 작업

  • docs/orchestration/을 단순 인터페이스가 아니라 재사용 가능한 개인 개발 문서 플러그인으로 정리했습니다.
  • 홈페이지에서 선택한 프로젝트에 기본 구조를 생성하거나, 기존 프로젝트용 마이그레이션 MD만 만들 수 있게 했습니다.

진행 내용

  • 문서 플러그인 버튼은 선택 프로젝트에 interface/, state/, reports/, templates/ 기본 구조를 생성합니다.
  • 마이그레이션 MD 버튼은 기존 문서를 새 규약으로 옮기기 위한 MIGRATION_PROMPT.md만 생성합니다.
  • 개발일지 목록은 날짜별 reports/YYYYMMDD/index.html만 우선 표시하도록 바꿨습니다.
  • 세부 unit HTML은 Discord 첨부나 상세 링크용으로 유지됩니다.

결과

  • 다른 프로젝트도 Project Orchestrator 홈페이지에서 기본 개발 문서 플러그인을 만들 수 있는 흐름이 생겼습니다.
  • LETHE처럼 이미 규약을 적용한 프로젝트는 날짜별 개발일지 중심으로 더 깔끔하게 표시됩니다.
  • 중앙 Discord 전송은 계속 Project Orchestrator가 담당하고, 프로젝트는 보고서 경로나 payload만 넘기는 구조를 유지합니다.
  • npm run build, 설치 dry-run, 대시보드 생성, API dry-run 검증이 통과했습니다.
