interface/state 구조와 중앙 Discord 전송

2026-06-08 작업 완료

어떤 작업

  • LETHE의 새 오케스트레이션 구조인 interface/, state/, 날짜별 reports/를 오케스트레이터가 읽도록 바꿨습니다.
  • LETHE가 작업 완료 후 Project Orchestrator에 보고 요청을 보내면, 오케스트레이터가 중앙 Discord 웹훅으로 대신 보내는 흐름을 추가했습니다.

진행 내용

  • 홈페이지 대시보드/다음 지시/런북 iframe은 docs/orchestration/interface/*.html을 먼저 읽습니다.
  • 스냅샷 문서 요약은 docs/orchestration/state/*.md를 먼저 읽습니다.
  • 개발일지는 reports/YYYYMMDD/index.html과 reports/YYYYMMDD/units/*.html을 재귀로 찾습니다.
  • POST /api/orchestration/discord-report는 LETHE가 보낸 projectId, reportPath, 또는 보고서 본문을 embed와 첨부로 변환합니다.

결과

  • npm run build가 통과했습니다.
  • LETHE의 interface/index.html과 interface/command.html이 API로 정상 반환됐습니다.
  • LETHE 최신 unit report가 개발일지 목록에 표시됐습니다.
  • LETHE가 호출할 intake API dry-run에서 embed payload와 HTML attachment metadata가 생성됐습니다.
