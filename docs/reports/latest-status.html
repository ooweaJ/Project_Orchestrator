어떤 작업

  • 대시보드에서 프로젝트를 직접 추가할 수 있게 만들었습니다.
  • 프로젝트 카드에서 삭제 확인 후 목록에서 제거할 수 있게 했습니다.
  • Codex 프롬프트 타입을 diagnose, commit, docs, review로 선택할 수 있게 했습니다.
  • 이번 작업 내용을 상태 문서, 다음 작업 문서, 개발 로그에 반영했습니다.

진행 내용

  • 테스트 프로젝트를 API로 추가하고 review 프롬프트를 생성한 뒤 삭제했습니다.
  • API 테스트 과정에서 data/projects.json 포맷이 자동 변경되어 원래 스타일로 정리했습니다.
  • 브라우저 시각 검증 대신 빌드와 API 호출로 기능을 검증했습니다.

결과

  • npm run build가 통과했습니다.
  • 프로젝트 추가/삭제 API 흐름이 정상 동작했습니다.
  • POST /api/projects/:id/prompt로 review 프롬프트 생성이 확인됐습니다.
  • 다음 작업은 최근 파일, TODO/FIXME, 대용량 파일 같은 스캐너 신호 강화입니다.
