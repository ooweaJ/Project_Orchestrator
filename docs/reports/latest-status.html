AI Project Orchestrator 진행 상태

2026-06-05 기준
현재 단계 폴더 선택
기본 프로젝트 2개
검증 Build/API

어떤 작업

  • 프로젝트 추가 폼에 찾아보기 버튼을 추가했습니다.
  • 로컬 드라이브와 하위 폴더를 대시보드 안에서 탐색할 수 있게 만들었습니다.
  • 원하는 폴더를 선택하면 프로젝트 경로 입력칸에 자동으로 들어가게 했습니다.
  • 백엔드에는 읽기 전용 폴더 목록 API인 GET /api/folders를 추가했습니다.

진행 내용

  • 브라우저 기본 보안상 일반 웹 입력으로는 절대경로를 직접 가져오기 어렵기 때문에 로컬 백엔드 기반 탐색기로 구현했습니다.
  • 드라이브 루트, 상위 폴더, 하위 폴더 이동을 지원하도록 했습니다.
  • 사용자 프로젝트 목록 파일 변경은 로컬 데이터로 보고 커밋에 포함하지 않았습니다.

결과

  • npm run build가 통과했습니다.
  • GET /api/folders가 로컬 드라이브와 폴더 목록을 반환했습니다.
  • GET /api/folders?path=<local-path>가 선택한 경로의 하위 폴더를 반환했습니다.
  • 대시보드는 http://127.0.0.1:5173에서 바로 확인할 수 있습니다.
  • 다음 작업은 실제 LETHE/SoulLike 경로를 연결해서 진짜 프로젝트 상태를 보는 것입니다.
