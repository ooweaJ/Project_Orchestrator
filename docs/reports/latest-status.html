어떤 작업

  • docs/orchestration/* 표준을 LETHE 피드백 기준으로 확장했습니다.
  • STATUS.md, RUNBOOK.md, SCOPE_GUARD.md, devlog/, reports/를 필수 코어에 포함했습니다.
  • AI 리뷰 기록, 증거, 템플릿 폴더는 권장 확장으로 분리했습니다.
  • 공통 AGENTS.md 양식은 먼저 템플릿으로 두고, 반복 검증 후 Codex skill로 승격하는 방향으로 정했습니다.
  • 다른 프로젝트에 기본 구조를 설치하는 npm run orchestration:install 명령을 추가했습니다.

진행 내용

  • STATUS.md는 전체 프로젝트 상태, CURRENT_TASK.md는 현재 작업 단위로 역할을 분리했습니다.
  • NEXT_TASKS.md는 상위 후보 중심으로 유지하고, 히스토리는 devlog/, 결정은 DECISION_LOG.md, 증거는 evidence/로 보내도록 정리했습니다.
  • 기존 루트 문서들은 바로 삭제하지 않고 호환 문서로 유지하는 방식을 선택했습니다.

결과

  • docs/ORCHESTRATION_INTERFACE.md가 새 표준을 설명합니다.
  • docs/orchestration/ 아래에 이 저장소의 초기 스캐폴드가 생겼습니다.
  • AGENTS.md가 새 오케스트레이션 문서 흐름을 안내합니다.
  • npm run orchestration:install -- --target . --dry-run 검증이 통과했습니다.
  • 다음 작업은 대시보드에서 이 인터페이스의 존재 여부를 감지하고 누락 파일을 스캐폴드하는 기능입니다.
