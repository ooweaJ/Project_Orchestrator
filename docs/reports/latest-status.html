어떤 작업

  • 선택된 프로젝트 상세 화면을 작업 지휘 중심으로 재구성했습니다.
  • 현재 작업, 진행 상태, 다음 작업 후보가 먼저 보이도록 했습니다.
  • 사용자가 직접 명령을 입력하면 현재 상태와 합쳐진 Codex 프롬프트를 만들 수 있게 했습니다.
  • 파일 신호는 접히는 보조 스캔 정보로 내려서 화면 우선순위를 낮췄습니다.

진행 내용

  • 프롬프트 메뉴가 보이지 않는 경우는 선택된 프로젝트가 없을 때라서, 빈 상태 안내를 더 명확하게 바꿨습니다.
  • 프롬프트 메뉴는 템플릿이고, 최종 명령은 사용자가 입력한 문장과 프로젝트 상태를 합쳐 만드는 구조로 바꿨습니다.
  • 사용자 프로젝트 목록 파일 변경은 로컬 데이터로 보고 커밋에 포함하지 않았습니다.

결과

  • npm run build가 통과했습니다.
  • 대시보드 오른쪽 패널에서 현재 작업과 다음 작업을 먼저 볼 수 있습니다.
  • 직접 입력한 명령을 포함한 Codex 프롬프트를 생성하고 복사할 수 있습니다.
  • 대시보드는 http://127.0.0.1:5173에서 바로 확인할 수 있습니다.
  • 다음 작업은 실제 LETHE/SoulLike 경로를 연결해서 진짜 프로젝트 상태를 보는 것입니다.
